功能富集與通路

ORA vs GSEA、clusterProfiler、Reactome、Decoupler — 把基因清單翻譯成生物學意義。

ORA vs. GSEA, clusterProfiler, Reactome, Decoupler — translate gene lists into biology.

兩大策略

一、ORA vs. GSEA

🎯 ORA

取一個固定的「重要基因清單」（如 padj < 0.05 的 DEG），用 hypergeometric / Fisher 檢驗其與 gene set 的重疊。優：簡單、直觀；缺：依賴主觀閾值、忽略 fold change 大小。

Take a fixed "important gene list" (e.g., padj < 0.05 DEGs); test overlap with gene sets via hypergeometric / Fisher. Pro: simple, intuitive; Con: threshold-dependent, ignores fold-change magnitude.

📊 GSEA

使用所有基因的排名 score，計算 gene set 在 ranking 上的聚集程度（running ES + permutation p-value）。優：無閾值、可偵測微小但協調的變化；缺：計算稍慢、結果對 ranking metric 敏感。

Uses all-gene ranking scores; tests how a gene set clusters in the ranking (running ES + permutation p). Pro: threshold-free, detects small coordinated shifts; Con: slower, sensitive to ranking metric.

資料庫與工具

二、資料庫與工具

📚 基因組註釋

GO (BP/MF/CC)、KEGG、Reactome、WikiPathways、MSigDB H/C2/C5、DoRothEA（TF→target）、PROGENy（pathway response）。

🛠️ R 工具

clusterProfiler（包含 enrichGO / enrichKEGG / GSEA）、fgsea（極快 GSEA）、GSVA（per-sample score）、ReactomePA、decoupleR（multi-method）。

🐍 Python 工具

GSEAPy、Enrichr (online + Python)、decoupler-py（單細胞友善）、Pathway Commons API、Omnipath。

單細胞 pathway 評分

AUCell、UCell、Vision、SCENIC（TF regulon）、VAM。可在 UMAP 上以連續顏色呈現 pathway activity。

多 omics 富集

mummichog（metabolome m/z）、FELLA（compound + KEGG 圖）、chromVAR（ATAC TF motif）、WMR（weighted multi-omics ranking）。

視覺化

cnetplot / emapplot（clusterProfiler）、EnrichmentMap（Cytoscape）、upset、ridgeplot、bubble plot。

決策

三、選擇指引

🌳 實用流程

Q1
有明確的「顯著基因清單」（DEG / DMR / 蛋白） → ORA 適合做首次快速 scan。
Q2
fold change 變化不大但成系統性方向（如老化、代謝重組） → GSEA 更敏感。
Q3
單細胞 cluster 想看每個 cluster 在每個 pathway 的活躍度 → 用 AUCell / UCell / decoupleR 的 per-cell score。
Q4
跨 omics 整合（RNA + Methyl + Metabolite）→ 用 decoupleR + OmniPath 共同 model 多層 signature；或 mummichog 直接跑 metabolite m/z。
Q1
Have a clear "significant gene list" (DEG/DMR/protein) → ORA for a fast first scan.
Q2
Small but coordinated fold-changes (aging, metabolic shift) → GSEA is more sensitive.
Q3
Per-cluster pathway activity in single-cell → AUCell / UCell / decoupleR per-cell scores.
Q4
Cross-omics integration (RNA + Methyl + Metabolite) → use decoupleR + OmniPath to jointly model multi-layer signatures; or mummichog directly on metabolite m/z.

五、實作

# clusterProfiler — ORA + GSEA
library(clusterProfiler); library(org.Hs.eg.db)

# ORA — KEGG
sig <- res$gene[res$padj < 0.05 & abs(res$log2FoldChange) > 1]
ora <- enrichKEGG(gene = sig, organism = "hsa", keyType = "ENTREZID")

# GSEA — Reactome
ranks <- res$log2FoldChange; names(ranks) <- res$gene
ranks <- sort(ranks, decreasing = TRUE)

library(ReactomePA)
gsea <- gsePathway(ranks, organism = "human", pvalueCutoff = 0.05)
dotplot(gsea, showCategory = 15)
emapplot(pairwise_termsim(gsea))
# fgsea — 極速 GSEA
library(fgsea); library(msigdbr)
h_gs <- msigdbr(species = "Homo sapiens", category = "H")
pathways <- split(h_gs$gene_symbol, h_gs$gs_name)

res_f <- fgsea(pathways = pathways, stats = ranks,
              minSize = 10, maxSize = 500, nproc = 8)
top <- res_f[order(padj)][1:20]
plotEnrichment(pathways[["HALLMARK_INFLAMMATORY_RESPONSE"]], ranks)
# decoupler-py — 跨方法、單細胞友善
import decoupler as dc
net = dc.get_progeny(organism = "human", top = 500)

# 跑多種方法
acts, _ = dc.run_mlm(mat=adata.X, net=net,
                       source="source", target="target", weight="weight")
adata.obsm["pathway_acts"] = acts

# 用 GSEA 風格做 ranking
dc.run_gsea(adata, net=net, source="source", target="target")
dc.pl.barplot(acts, top=20)
