CH 2 / 20 · BLOCK 1 FOUNDATIONS

實驗設計與功率分析

差的設計再強的統計也救不回來。本章涵蓋生物 vs. 技術重複、隨機化、blocking、batch confounding 與多種組學的樣本數估算。

No statistics can rescue a bad design. We cover biological vs. technical replicates, randomization, blocking, batch confounding, and sample-size estimation for multiple omics.

核心區分

一、生物重複 vs. 技術重複

🧬 生物重複

來自不同個體 / 不同生物樣本的測量。捕捉真正的群體變異，是統計推論的基礎。少於 3 個 biological replicates 幾乎沒辦法做 DE 分析。

Measurements from different individuals / biological specimens. Capture true population variation — the foundation of statistical inference. Fewer than 3 biological replicates makes DE analysis essentially impossible.

⚙️ 技術重複

同一樣本重新走流程（同一管 RNA 重複 library prep / sequencing）。只反映儀器與流程的雜訊，不能取代生物重複。現代 NGS 技術雜訊已很低，多數情況可省略。

Same sample re-run through the pipeline (same RNA tube, repeat library prep / sequencing). Captures only instrument/pipeline noise — cannot replace biological replicates. Modern NGS technical noise is low; often skippable.

⛔
常見錯誤：把同一個 mouse 的肝臟切成 3 塊送 RNA-seq，當作 n=3。這是「pseudoreplicates」，n 仍然 = 1，p-value 是假的。Hurlbert (1984) 已指出這類設計的問題。 Common mistake: Splitting one mouse liver into 3 pieces for RNA-seq and calling it n=3. These are pseudoreplicates — n is still 1, and p-values are inflated. Hurlbert (1984) called this out.
批次效應

二、批次混淆

如果所有「對照組」在週一定序、所有「實驗組」在週二定序，就無法區分條件效應與批次效應 — 任何下游 batch correction（Ch 14）都救不回來。預防勝於補救。

If all "controls" are sequenced on Monday and all "treated" on Tuesday, you cannot separate condition from batch — no downstream batch correction (Ch 14) will save you. Prevention beats remediation.

❌ 混淆設計

Plate 1：全部 control；Plate 2：全部 treated。

Plate 1: all controls; Plate 2: all treated.

⚖️ 平衡設計

每個 plate / 跑次都各放一半 control 一半 treated；統計上可分離批次效應。

Each plate / run has half controls and half treated — statistically separable from batch.

✅ 隨機 + Block

在每個 block (plate/lane) 內以隨機順序放入條件，並於模型中納入 block term。金標準。

Within each block (plate/lane) randomize condition order; include block as a model term. Gold standard.

五、設計檢查清單

🌳 送樣前 7 個問題

Q1
主要對比 (primary contrast) 寫得出來嗎？例：treated vs. untreated within tissue X at timepoint Y。
Q2
n 是生物重複還是 pseudoreplicate？每組 ≥ 3？
Q3
批次有沒有與條件混淆？每個 plate / lane 都 balanced？
Q4
樣本處理順序有 randomize 嗎？（操作員、時間、儀器位置）
Q5
已知會影響結果的 covariate（age、sex、RIN、batch）有記錄嗎？
Q6
樣本量是否足以達到 80% power？做了 pilot 或文獻估計？
Q7
有預先註冊（pre-registration）或 lock-down 分析計畫嗎？
Q1
Can you state the primary contrast? e.g., treated vs. untreated within tissue X at timepoint Y.
Q2
Is n biological or a pseudoreplicate? Is each group ≥ 3?
Q3
Are batches confounded with condition? Each plate / lane balanced?
Q4
Is processing order randomized (operator, time, instrument position)?
Q5
Are known covariates (age, sex, RIN, batch) recorded?
Q6
Does sample size give ≥ 80% power? Based on pilot or literature?
Q7
Is the analysis plan pre-registered or locked down?
程式碼

六、功率計算實作

# R：RNA-seq 功率計算 / RNA-seq power
library(RNASeqPower)

# cv 取自先導實驗或文獻；alpha=0.05 ÷ 估計 DE 基因數做粗略校正
rnapower(depth = 20,      # 平均 reads/gene
          n = 5,            # 每組樣本數
          cv = 0.4,         # 生物 CV
          effect = 2,        # fold change
          alpha = 5e-6)      # 校正後 alpha

# 通用 two-sample t-test power / Generic t-test power
library(pwr)
pwr.t.test(d = 0.8, power = 0.8,
           sig.level = 0.05, type = "two.sample")
# d = Cohen's d = (μ1-μ2)/σ；輸出 n/group
# Python：statsmodels TTestIndPower / power.solve_power
from statsmodels.stats.power import TTestIndPower

ana = TTestIndPower()
n = ana.solve_power(effect_size=0.8, alpha=0.05, power=0.8)
print(f"n/group: {n:.1f}")

# scRNA-seq 細胞數估算（粗略）/ scRNA cell-count rule of thumb
# 想偵測佔比 p 的稀有族群、每族群至少 m 個細胞，每樣本至少 m/p 個細胞
p, m = 0.01, 50
print(f"Cells / sample >= {m/p:.0f}")   # 5000 cells

# GWAS：基於 OR 與 MAF 估 n
from statsmodels.stats.proportion import proportions_ztest
# 實務上用 GAS Power Calculator 或 Quanto
💡
實用建議：用 GTEx / TCGA / HCA 等公開資料抓 CV 與基線分佈，比憑空猜更靠譜。Power 是 effect size、樣本數、雜訊三者拉鋸 — 弄清楚最低可接受 fold change 比追求 80% 還重要。 Practical tip: Pull CV and baselines from GTEx / TCGA / HCA rather than guessing. Power is a tug-of-war between effect size, n, and noise — knowing your minimum interesting fold change matters more than chasing 80%.
