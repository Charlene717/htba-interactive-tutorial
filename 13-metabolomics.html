CH 13 / 20 · BLOCK 4 METABOLOMICS

代謝體

untargeted vs targeted、XCMS、MS-DIAL、MetaboAnalyst — 從 m/z–RT 表到生物學意義。

Untargeted vs. targeted, XCMS, MS-DIAL, MetaboAnalyst — from m/z–RT tables to biology.

兩大策略

一、兩大策略

🌐 Untargeted

全面 scan 所有 m/z，發現未知 feature。產出：m/z × RT × intensity 表，常數千至數萬 feature。儀器：Orbitrap、QToF、IM-MS。挑戰：多數 feature 無法鑑定（identification rate 常 < 20%）。

Scan all m/z to discover unknown features. Output: m/z × RT × intensity table — thousands to tens of thousands of features. Instruments: Orbitrap, QToF, IM-MS. Challenge: most features remain unidentified (ID rate often < 20%).

🎯 Targeted

預先選擇 10-300 個已知代謝物，用 MRM/PRM 高精度量化。線性範圍寬、定量準確，是臨床指標物質的金標準。

Pre-select 10–300 known metabolites; quantify with high precision via MRM/PRM. Wide linear range, accurate quantification — gold standard for clinical biomarkers.

流程

二、分析流程

採樣與前處理

紀錄 quench/extract 條件、加 internal standards、隨機化注射順序、每 10 樣本插 pooled QC、blank。

Record quench/extract conditions, add internal standards, randomize injection order, insert pooled QC every 10 samples plus blanks.

峰偵測 + 校正

XCMS（R 經典）、MS-DIAL（GUI）、MZmine 3、asari（speed champion 2023）。輸出 feature table。

XCMS (classic R), MS-DIAL (GUI), MZmine 3, asari (speed champion 2023). Output: feature table.

QC + normalization

用 pooled QC 監控 RT drift 與 intensity drift；常用 SERRF、QC-RLSC、NOMIS 校正 batch effect。

Use pooled QC to track RT and intensity drift; SERRF, QC-RLSC, NOMIS for batch correction.

Annotation / ID

資料庫：HMDB v5、METLIN、GNPS、MoNA。MSI 信賴度：L1 (純品標準)、L2 (譜庫匹配)、L3 (類別)、L4 (未知)。深度學習：SIRIUS 5 + CSI:FingerID、MS2DeepScore。

Databases: HMDB v5, METLIN, GNPS, MoNA. MSI levels: L1 (authentic standard), L2 (library match), L3 (class), L4 (unknown). DL: SIRIUS 5 + CSI:FingerID, MS2DeepScore.

統計與通路

MetaboAnalyst 6、POMA、limma。Pathway：mummichog（從 m/z 直接 pathway）、KEGG / Reactome 整合。

MetaboAnalyst 6, POMA, limma. Pathway: mummichog (m/z → pathway directly), KEGG / Reactome integration.

# XCMS peak picking → alignment → fill
library(xcms); library(MSnbase)
files <- list.files("mzML/", full.names=TRUE)
raw  <- readMSData(files, mode="onDisk",
                  pdata=new("NAnnotatedDataFrame", pheno))

cwp  <- CentWaveParam(ppm=15, peakwidth=c(5,25), snthresh=10)
xset <- findChromPeaks(raw, param=cwp)
xset <- adjustRtime(xset, param=PeakGroupsParam())
xset <- groupChromPeaks(xset, param=PeakDensityParam(sampleGroups=pheno$group))
xset <- fillChromPeaks(xset)

mat <- featureValues(xset, value="into")
# MetaboAnalystR — 統計 + pathway
library(MetaboAnalystR)
mSet <- InitDataObjects("pktable", "stat", FALSE)
mSet <- Read.TextData(mSet, "feature_table.csv", "rowu", "disc")
mSet <- SanityCheckData(mSet) |>
        ReplaceMin() |> Normalization("NULL", "LogNorm", "ParetoNorm")

mSet <- PCA.Anal(mSet)
mSet <- PLSR.Anal(mSet)
mSet <- Volcano.Anal(mSet, fcthresh=2, alpha=0.05)
# Pathway from m/z 直接（mummichog）
mSet <- PerformMummichog(mSet, "hsa_kegg", 10, "negative")
# matchms / SIRIUS — Python spectral matching
from matchms.importing import load_from_msp
from matchms.similarity import CosineGreedy
from matchms import calculate_scores

queries = list(load_from_msp("queries.msp"))
refs    = list(load_from_msp("hmdb_lib.msp"))
scores  = calculate_scores(refs, queries, CosineGreedy(tolerance=0.01))

for q in queries[:5]:
    top = scores.scores_by_query(q, sort=True)[:3]
    for r, s in top: print(r.metadata["compound_name"], s)
