FASTQ → counts 上游流程

所有組學分析的共同地基。QC、trimming、比對、量化、報告整合 — 用 nf-core 一鍵跑完。

The common bedrock for all omics. QC, trimming, alignment, quantification, reporting — run end-to-end with nf-core.

總覽

一、上游流程拆解

FastQC + MultiQC

原始 reads 的 per-base quality、adapter content、duplication。MultiQC 將數十個樣本 QC 摘要彙整成一頁 HTML。

Per-base quality, adapter content, duplication of raw reads. MultiQC aggregates dozens of samples into one HTML report.

修剪

Trimmomatic、cutadapt、fastp（推薦：快、整合 QC、UMI 支援）、Trim Galore。經驗法則：Q20 滑動視窗 + 移除過短 reads (<36 bp)。

Trimmomatic, cutadapt, fastp (recommended: fast, integrated QC, UMI support), Trim Galore. Rule of thumb: Q20 sliding window + drop reads <36 bp.

比對

DNA: bwa-mem2、minimap2（長讀）；RNA: STAR、HISAT2；pseudoalignment: Salmon、kallisto；scRNA: STARsolo、Cell Ranger、alevin-fry。

DNA: bwa-mem2, minimap2 (long-read); RNA: STAR, HISAT2; pseudoalignment: Salmon, kallisto; scRNA: STARsolo, Cell Ranger, alevin-fry.

後處理

samtools sort、Picard / GATK MarkDuplicates、GATK BQSR（DNA）、splice-aware merge（RNA）。產出 analysis-ready BAM/CRAM。

samtools sort, Picard / GATK MarkDuplicates, GATK BQSR (DNA), splice-aware merge (RNA). Outputs analysis-ready BAM/CRAM.

量化

featureCounts、HTSeq、Salmon (transcript-level)、HaplotypeCaller、DeepVariant、MACS2（peaks）、Bismark methylation extractor。

featureCounts, HTSeq, Salmon (transcript-level), HaplotypeCaller, DeepVariant, MACS2 (peaks), Bismark methylation extractor.

三、Phred score 視覺化

調整 Q 值看錯誤率 P：P = 10−Q/10。Q20 表示 1% 錯誤、Q30 表示 0.1% 錯誤；trimming 常用閾值 Q20–Q30。

Move Q to see error probability P = 10−Q/10. Q20 = 1% error, Q30 = 0.1%; common trimming thresholds are Q20–Q30.

P = 10−Q/10 = 0.001 = 0.1% error
管線

四、nf-core 一鍵管線

nf-core 是 community-curated Nextflow 管線集，每個流程都有 CI 測試、Docker/Singularity 容器、版本標籤與 ENA/SRA 整合。

常用管線：

  • nf-core/rnaseq — Bulk RNA-seq (Salmon + STAR + featureCounts + MultiQC)
  • nf-core/scrnaseq — Cell Ranger / STARsolo / alevin-fry
  • nf-core/sarek — WGS/WES germline + somatic（GATK/DeepVariant/Strelka）
  • nf-core/atacseq、nf-core/chipseq — peak calling 全流程
  • nf-core/methylseq — Bismark / bwa-meth
  • nf-core/fetchngs — 自動從 SRA/ENA/GEO 抓資料 + metadata

nf-core is the community-curated Nextflow pipeline collection — every workflow has CI tests, Docker/Singularity containers, versioned releases, and ENA/SRA integration.

Popular pipelines:

  • nf-core/rnaseq — Bulk RNA-seq (Salmon + STAR + featureCounts + MultiQC)
  • nf-core/scrnaseq — Cell Ranger / STARsolo / alevin-fry
  • nf-core/sarek — WGS/WES germline + somatic (GATK/DeepVariant/Strelka)
  • nf-core/atacseq, nf-core/chipseq — end-to-end peak calling
  • nf-core/methylseq — Bismark / bwa-meth
  • nf-core/fetchngs — auto-fetch from SRA/ENA/GEO with metadata
💡
替代品：Snakemake-Wrappers、Galaxy、Cromwell + WDL（Broad / Terra 生態）。研究團隊請優先採用社群管線而非自寫 — 重現性與升級成本差異巨大。 Alternatives: Snakemake-Wrappers, Galaxy, Cromwell + WDL (Broad / Terra). Prefer community pipelines over hand-rolled — the gap in reproducibility and upgrade cost is huge.
程式碼

五、完整 RNA-seq 上游

# 1. QC + trim
fastp -i sample_R1.fq.gz -I sample_R2.fq.gz \
      -o trim_R1.fq.gz -O trim_R2.fq.gz \
      -q 20 -l 36 -j sample.json -h sample.html

# 2. STAR 比對（2-pass）
STAR --runThreadN 16 --genomeDir GRCh38_index/ \
     --readFilesIn trim_R1.fq.gz trim_R2.fq.gz \
     --readFilesCommand zcat --twopassMode Basic \
     --outSAMtype BAM SortedByCoordinate \
     --outFileNamePrefix sample.

# 3. 計數
featureCounts -T 16 -a GRCh38.gtf -o counts.txt \
            -p --countReadPairs sample.Aligned.sortedByCoord.out.bam

# 4. MultiQC 報告
multiqc .
# nf-core/rnaseq — 一個指令跑完所有步驟
# 安裝：conda install -c bioconda nextflow nf-core
nextflow run nf-core/rnaseq \
    -r 3.16.0 \
    --input samplesheet.csv \
    --outdir results/ \
    --genome GRCh38 \
    --aligner star_salmon \
    -profile docker \
    -resume

# samplesheet.csv 範例：
# sample,fastq_1,fastq_2,strandedness
# CTRL_1,ctrl1_R1.fq.gz,ctrl1_R2.fq.gz,reverse
# CTRL_2,ctrl2_R1.fq.gz,ctrl2_R2.fq.gz,reverse
# TRT_1,trt1_R1.fq.gz,trt1_R2.fq.gz,reverse
# Snakefile — 用 wildcards 自動展開到所有樣本
SAMPLES = ["CTRL_1", "CTRL_2", "TRT_1", "TRT_2"]

rule all:
    input: "results/counts.tsv", "results/multiqc_report.html"

rule fastp:
    input: r1="raw/{sample}_R1.fq.gz", r2="raw/{sample}_R2.fq.gz"
    output: r1="trim/{sample}_R1.fq.gz", r2="trim/{sample}_R2.fq.gz",
            j="qc/{sample}.json"
    shell: "fastp -i {input.r1} -I {input.r2} -o {output.r1} -O {output.r2} -j {output.j}"

rule star:
    input: r1="trim/{sample}_R1.fq.gz", r2="trim/{sample}_R2.fq.gz"
    output: "bam/{sample}.bam"
    threads: 16
    shell: "STAR --runThreadN {threads} --genomeDir idx/ "
           "--readFilesIn {input.r1} {input.r2} --readFilesCommand zcat "
           "--outSAMtype BAM SortedByCoordinate "
           "--outFileNamePrefix bam/{wildcards.sample}."
# … rule featurecounts, rule multiqc 略
