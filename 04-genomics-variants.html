變異偵測與註釋

從 BAM 到 VCF：GATK best practices、DeepVariant、VEP、ClinVar 致病性註釋 — germline 與 somatic 一次涵蓋。

From BAM to VCF: GATK best practices, DeepVariant, VEP, ClinVar pathogenicity — covering both germline and somatic.

總覽

一、變異呼叫的兩條主線

🧬 Germline

來自卵 / 精子，全身細胞共有，符合 Mendelian 機率。主要工具：GATK HaplotypeCaller、DeepVariant（Google）。常做 joint genotyping（GVCF + GenotypeGVCFs）。下游：罕見病、carrier screen、GWAS。

From egg/sperm, shared by all cells, follows Mendelian frequencies. Tools: GATK HaplotypeCaller, DeepVariant (Google). Joint genotyping (GVCF + GenotypeGVCFs). Downstream: rare disease, carrier screen, GWAS.

🎯 Somatic

後天突變，腫瘤組織與正常組織配對比較。主要工具：GATK Mutect2、Strelka2、VarScan2、Lancet（複雜區域）。VAF 可低至 1% 需高深度（>100×）。下游：driver mutation、TMB、neoantigen。

Acquired mutations, tumor-vs-normal paired. Tools: GATK Mutect2, Strelka2, VarScan2, Lancet (complex regions). VAF as low as 1% — needs high depth (>100×). Downstream: drivers, TMB, neoantigens.

GATK 流程

二、GATK Best Practices 流程

BWA-MEM2 alignment

用 read group (-R "@RG\tID:...\tSM:...\tPL:ILLUMINA") 標註樣本，後續 BQSR 與 joint genotyping 才能跑。

Tag with read group (-R "@RG\tID:...\tSM:...\tPL:ILLUMINA") — required for BQSR and joint genotyping.

MarkDuplicates

Picard / GATK MarkDuplicates — 移除 PCR 重複。UMI-based library 改用 fgbio CallDuplexConsensusReads。

Picard / GATK MarkDuplicates — remove PCR duplicates. For UMI libraries use fgbio CallDuplexConsensusReads.

BQSR (Base Quality Score Recalibration)

用 dbSNP / Mills indels 做機器學習，校正系統性 Q 值偏差。長讀資料不需要。

Machine-learning recalibration using dbSNP / Mills indels — corrects systematic Q-score bias. Skip for long reads.

HaplotypeCaller → GVCF

每個樣本獨立呼叫，輸出 GVCF（含 reference block）。為 cohort joint genotyping 做準備。

Per-sample calling outputs GVCF (with reference blocks). Sets up cohort joint genotyping.

GenotypeGVCFs + VQSR

Joint 處理整個 cohort，再用 VQSR（Gaussian mixture model）過濾低品質 variant；小 cohort 改用 hard filter。

Joint over whole cohort, then VQSR (Gaussian mixture) filters low-quality variants; for small cohorts use hard filters.

⚠️
DeepVariant 替代：Google 的 CNN-based 呼叫器（PrecisionFDA 多次奪冠），不需 BQSR、不需 VQSR，但對於 cohort joint analysis 仍需 GLnexus 後處理。長讀（HiFi、ONT）有專屬模型。 DeepVariant alternative: Google's CNN caller (multiple PrecisionFDA wins) skips BQSR / VQSR, but cohort joint analysis still needs GLnexus. Long-read models exist for HiFi and ONT.
變異註釋

三、變異註釋

📄 功能影響

VEP（Ensembl）、ANNOVAR、snpEff：把 chr-pos-ref-alt 翻成 missense / synonymous / stop_gained 等 SO 詞彙，並對應基因 / transcript / protein 改變。

VEP (Ensembl), ANNOVAR, snpEff: translate chr-pos-ref-alt into SO terms (missense / synonymous / stop_gained) plus gene/transcript/protein consequences.

📊 族群頻率

gnomAD v4（807k exomes + 76k genomes）：取得 AF、AC、AN，並提供 popmax；rare variant 通常定義 AF < 0.1% 或 < 1%。

gnomAD v4 (807k exomes + 76k genomes): AF, AC, AN, popmax. Rare variants typically defined as AF < 0.1% or < 1%.

🩺 臨床顯著性

ClinVar：人工策展，5 級分類（B / LB / VUS / LP / P）。InterVar、Franklin：依 ACMG/AMP 2015 準則自動化評分。

ClinVar: human-curated 5-tier (B / LB / VUS / LP / P). InterVar, Franklin: automate ACMG/AMP 2015 criteria.

🤖 In silico 預測

AlphaMissense (DeepMind 2023)、REVEL、CADD、SpliceAI。新版 ACMG 建議用 calibrated probabilities，而非單一 cutoff。

AlphaMissense (DeepMind 2023), REVEL, CADD, SpliceAI. Newer ACMG guidance prefers calibrated probabilities over a single cutoff.

💊 藥物基因組

PharmGKB / CPIC：CYP2D6、CYP2C19、TPMT 等代謝酶基因型 → 處方建議。Stargazer / PharmCAT 自動分型。

PharmGKB / CPIC: CYP2D6, CYP2C19, TPMT genotype → prescribing guidance. Stargazer / PharmCAT for automated star-allele typing.

🧠 GWAS 後續

用 FUMA、LocusZoom、OpenTargets Genetics 解讀 hit 的功能與基因；Fine-mapping 用 SuSiE / CAVIAR。

Use FUMA, LocusZoom, OpenTargets Genetics to interpret hits; fine-mapping via SuSiE / CAVIAR.

互動 — ACMG 分類

四、ACMG 分類模擬器

勾選證據，依 Richards et al. 2015 規則計算最終分類。注意：僅為教學示意，臨床請用 InterVar / Franklin 並由 board-certified geneticist 審核。

Check evidence, see final classification per Richards et al. 2015. Note: teaching only — clinical use requires InterVar / Franklin reviewed by board-certified geneticists.

Score: P=0, B=0 VUS

五、實作

# Germline single-sample → joint cohort
gatk HaplotypeCaller \
    -R Homo_sapiens.GRCh38.fa \
    -I sample.dedup.bqsr.bam \
    -O sample.g.vcf.gz \
    -ERC GVCF

gatk CombineGVCFs -R ref.fa \
    -V s1.g.vcf.gz -V s2.g.vcf.gz -V s3.g.vcf.gz \
    -O cohort.g.vcf.gz
gatk GenotypeGVCFs -R ref.fa -V cohort.g.vcf.gz -O cohort.vcf.gz

# VQSR（SNP）
gatk VariantRecalibrator -V cohort.vcf.gz \
    --resource:hapmap,known=false,training=true,truth=true,prior=15 hapmap.vcf \
    --resource:omni,known=false,training=true,truth=true,prior=12 omni.vcf \
    -an QD -an MQ -an FS -an SOR -mode SNP \
    -O snp.recal --tranches-file snp.tranches
# DeepVariant 一鍵呼叫（Docker）
BIN_VERSION="1.6.1"
docker run --gpus all \
  -v "$PWD":/input -v "$PWD":/output \
  google/deepvariant:${BIN_VERSION}-gpu \
  /opt/deepvariant/bin/run_deepvariant \
    --model_type=WGS \
    --ref=/input/ref.fa \
    --reads=/input/sample.bam \
    --output_vcf=/output/sample.vcf.gz \
    --num_shards=16

# 長讀（PacBio HiFi）
--model_type=PACBIO  # 改這行即可
# R 註釋與篩選 / R annotation and filtering
library(VariantAnnotation); library(GenomicFeatures)
library(TxDb.Hsapiens.UCSC.hg38.knownGene)

vcf <- readVcf("cohort.vcf.gz", "hg38")
txdb <- TxDb.Hsapiens.UCSC.hg38.knownGene

# 取 high-impact missense / stop-gain
coding <- predictCoding(vcf, txdb, seqSource=FaFile("ref.fa"))
miss   <- coding[coding$CONSEQUENCE %in% c("nonsynonymous", "stop")]

# 與 ClinVar 對接
library(myvariant)
ann <- getVariants(names(miss),
                   fields = "clinvar.rcv.clinical_significance,gnomad_exome.af")
