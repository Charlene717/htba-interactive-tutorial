生物基礎模型

scGPT、Geneformer、ESM-3、AlphaFold 3、Tahoe-100M — 它們能做什麼，又有哪些誇大？

scGPT, Geneformer, ESM-3, AlphaFold 3, Tahoe-100M — what they can do, and where the hype falls short.

基本概念

一、概念

定義：在大規模生物資料（千萬至億級樣本）以 self-supervised 預訓練的大型 Transformer / GNN，可遷移到多個下游任務（cell type、perturbation response、protein function、drug-target、structure prediction）。

2023-2025 是「scaling laws 進入生醫」的年代。文獻指出：

  • 有些 zero-shot 任務 foundation model 不如傳統 ML（Boiarsky et al. 2024、Microsoft 2025）— 大模型 ≠ 自動更好；
  • 但在 perturbation prediction、cross-species transfer、low-data fine-tune 場景上，預訓練優勢明確。

Definition: large Transformers / GNNs pre-trained self-supervised on massive biological corpora (tens of millions to billions of samples), transferable to many downstream tasks (cell type, perturbation response, protein function, drug-target, structure prediction).

2023-2025 is the era of "scaling laws meet biology." Literature notes:

  • For some zero-shot tasks, foundation models underperform classical ML (Boiarsky et al. 2024, Microsoft 2025) — bigger ≠ automatically better;
  • But for perturbation prediction, cross-species transfer, and low-data fine-tuning, pretraining advantages are clear.

現實檢驗

三、別買 hype

⚠️
2024-2025 benchmark 大量警告：
• Boiarsky et al. (Microsoft Research, 2024)：scGPT / Geneformer 在 zero-shot batch integration 表現不如 Harmony / scVI，甚至不如 HVG。
• Liu et al. Adv Sci 2025：foundation models 在 cell-type annotation 上有時不如 scANVI；在 perturbation 與 cross-species transfer 才看到真正優勢。
• 2025 mechanistic 研究：scGPT / Geneformer 學到的多是 co-expression / pathway 結構，不是因果調控。 Reality from 2024–2025 benchmarks:
• Boiarsky et al. (MSR 2024): scGPT / Geneformer trail Harmony / scVI on zero-shot batch integration — sometimes worse than HVG alone.
• Liu et al. Adv Sci 2025: foundation models can be beaten by scANVI on cell-type annotation; their real edge appears in perturbation and cross-species transfer.
• 2025 mechanistic studies: scGPT / Geneformer mostly learn co-expression / pathway structure — not causal regulatory logic.
💡
Best practice：用 foundation model 做特徵抽取（embedding），但統計分析仍用傳統 DESeq2 / Wilcoxon；fine-tuning 一定要做 hold-out test；report 與傳統方法的比較。 Best practice: Use FMs for feature extraction (embeddings), but keep statistical inference with DESeq2 / Wilcoxon; always hold out a test set when fine-tuning; report comparisons against classical methods.

五、實作

# scGPT — zero-shot embedding + fine-tune
import scgpt
adata = scgpt.tasks.embed_data(
    adata, model_dir="./scgpt_human_pretrained",
    batch_size=64, use_fast_transformer=True)

# 把 embedding 當 PCA 用
sc.pp.neighbors(adata, use_rep="X_scGPT")
sc.tl.umap(adata); sc.tl.leiden(adata, resolution=0.5)

# 比較與 Harmony / scVI（一定要做！）
from scib_metrics.benchmark import Benchmarker
bm = Benchmarker(adata, batch_key="batch", label_key="cell_type",
                 embedding_obsm_keys=["X_scGPT", "X_pca_harmony"])
bm.benchmark(); bm.plot_results_table()
# Geneformer — in silico perturbation
from geneformer import EmbExtractor, InSilicoPerturber

emb = EmbExtractor(model_type="CellClassifier", num_classes=10,
                   filter_data={"cell_type": ["T cell"]})
emb.extract_embs("./Geneformer-V2-316M", "./tokenized.dataset",
                  "./embs/", "output_prefix")

# In silico KO + 觀察 expression 改變
isp = InSilicoPerturber(perturb_type="delete",
                        perturb_rank_shift=None, genes_to_perturb=["FOXP3"],
                        cell_states_to_model={"state_key":"cell_type"})
isp.perturb_data("./Geneformer-V2-316M", "./tokenized.dataset", "./out/")
# ESM-3 — 蛋白序列 → 結構 + 功能
# pip install esm
from esm.models.esm3 import ESM3
from esm.sdk.api import ESMProtein, GenerationConfig

model = ESM3.from_pretrained("esm3_sm_open_v1").to("cuda")
prot  = ESMProtein(sequence="MKTAYIAKQRQISFVK...")

# 預測結構 (PDB)
out = model.generate(prot, GenerationConfig(track="structure", num_steps=8))
out.to_pdb("predicted.pdb")

# 也可給 partial structure / function 限制做 conditional generation
