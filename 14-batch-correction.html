CH 14 / 20 · BLOCK 5 INTEGRATION

批次效應與水平整合

同一組學 × 跨樣本 / 跨研究：ComBat、Harmony、scVI、Scanorama、MNN — benchmark 與選型。

Same omics × across samples / studies: ComBat, Harmony, scVI, Scanorama, MNN — benchmarks and selection.

核心觀念

一、為什麼需要

水平整合 (horizontal integration)：同一種 omics、跨樣本 / 跨研究合併（如：5 個 lab 的 scRNA-seq、TCGA + GTEx）。批次效應來自試劑批次、操作員、定序儀、年份等系統性差異，常與條件混淆，會在 PCA 第一軸主導變異 — 必須校正才能做 cohort 級別的分析。

核心 trade-off：「過度校正」會洗掉真實生物訊號（如疾病 vs. 正常）；「校正不足」則 PCA / DE 會被批次主宰。Benchmark（Tran 2020、Luecken 2022 NBT）已成為選型依據。

Horizontal integration: same omics, across samples / studies (e.g., scRNA-seq from 5 labs; TCGA + GTEx). Batch effects come from reagent lots, operators, instruments, years — systematic differences often confounded with condition. They dominate the first PCA axis and must be corrected for cohort-level analysis.

Core trade-off: over-correction wipes out real biological signal (e.g., disease vs. normal); under-correction lets batches dominate PCA / DE. Benchmarks (Tran 2020, Luecken 2022 NBT) guide selection.

評估

三、評估指標

good integration = mix batches（同細胞型態跨批次混在一起） + preserve biology（不同細胞型態保持分開）。常用：

  • kBET：批次混合的 K-test。
  • LISI（iLISI 看 batch mixing、cLISI 看 cell-type purity）。
  • ASW (Avg silhouette width)：以 cell-type 與 batch 為標籤分別計算。
  • scIB / scib-metrics（Luecken 2022 NBT）：整合 14 個指標，產生 overall score。

Good integration = mix batches (same cell types across batches blend) + preserve biology (different cell types stay apart). Standard metrics:

  • kBET: K-test for batch mixing.
  • LISI (iLISI for batch mixing, cLISI for cell-type purity).
  • ASW (Avg silhouette width) labeled by cell type or batch.
  • scIB / scib-metrics (Luecken 2022 NBT): aggregates 14 metrics into one score.

# Bulk RNA：ComBat-seq 直接吃 counts
library(sva)
adj <- ComBat_seq(counts, batch=pheno$batch, group=pheno$condition)
# 然後再進 DESeq2

# scRNA：Harmony with Seurat
library(Seurat); library(harmony)
merged <- merge(list_seurat_obj) |>
          NormalizeData() |> FindVariableFeatures() |>
          ScaleData() |> RunPCA() |>
          RunHarmony(group.by.vars="sample_id", theta=2)
merged <- RunUMAP(merged, reduction="harmony", dims=1:30) |>
          FindNeighbors(reduction="harmony", dims=1:30) |>
          FindClusters(algorithm=4)
# Scanpy + Harmony
import scanpy as sc, scanpy.external as sce
sc.pp.pca(adata, n_comps=50)
sce.pp.harmony_integrate(adata, key="sample_id", theta=2)
sc.pp.neighbors(adata, use_rep="X_pca_harmony")
sc.tl.umap(adata); sc.tl.leiden(adata, resolution=0.5)

# 評估
from scib_metrics.benchmark import Benchmarker
bm = Benchmarker(adata, batch_key="sample_id", label_key="cell_type",
                 embedding_obsm_keys=["X_pca", "X_pca_harmony"])
bm.benchmark(); bm.plot_results_table()
# scVI / scANVI — 大 atlas 主流
import scvi
scvi.model.SCVI.setup_anndata(adata, batch_key="sample_id",
                              continuous_covariate_keys=["pct_mito"])
model = scvi.model.SCVI(adata, n_layers=2, n_latent=30)
model.train(max_epochs=200)
adata.obsm["X_scVI"] = model.get_latent_representation()

# scANVI — 含 label transfer
scvi.model.SCANVI.setup_anndata(adata, batch_key="sample_id",
                                labels_key="cell_type", unlabeled_category="unknown")
scanvi_model = scvi.model.SCANVI(adata)
scanvi_model.train(max_epochs=100)
